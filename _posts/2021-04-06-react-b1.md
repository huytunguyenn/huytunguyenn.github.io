### **Stateful Component**

normal: mỗi lần state thay đổi thì phải redraw lại giao diện. ⇒ React ko cần redraw ⇒ nhờ ma thuật `setState`

- khi React thấy `setState` thay đổi ⇒ tự động gọi `render()`

```jsx
class Timer extends React.Component {
  constructor(props) {       
    super(props);    // gọi hàm dựng của lớp cha (tức là truyền props vô constructor của React.Component)
    this.state = { seconds: 0 }; // trạng thái đầu tiên gán = 0
  }

  tick() {  // mỗi s tăng lên 1
    this.setState(state => ({
      seconds: state.seconds + 1 
    }));
	// ko cần redraw :3
  }

  componentDidMount() { // mỗi s gọi hàm tick() 1 lần
    this.interval = setInterval(() => this.tick(), 1000); 
  }

  componentWillUnmount() {  
    clearInterval(this.interval); 
  }

  render() {
    return (
      <div>
        Seconds: {this.state.seconds}
      </div>
    );
  }
}

ReactDOM.render(
  <Timer />,
  document.getElementById('timer-example')
);
```

**Result**

```jsx
Seconds: 0 // mỗi s tự tăng lên 1
```

### **Thay đổi trạng thái**:

***vd1:***  *Muốn thay đổi màu?* 

- Không thể xài `<div style='color:red'>` được vì React đã thay đổi cái *style* chỉ nhận object {}
- Đối với JSX, nếu muốn truyền JS vào HTML thì phải bọc vô ngoặc nhọn {}

```jsx
render() {
    return ( // ngoặc trong để tạo object, ngoặc ngoài do muốn viết JS trong HTML
      <div style={{color:'blue'}}> 
        Seconds: {this.state.seconds}
      </div>
    );
}

```

Truyền vào component bằng `props`:

```jsx
render() {
    return (
      <div style={{color: this.props.color}}>  
        Seconds: {this.state.seconds}
      </div>
    );
}

ReactDOM.render(  // như truyền thuộc tính HTML => lưu trong props
  <Timer color='blue'/>,
  document.getElementById('timer-example')
);
```

### Event handlers

```jsx
class TodoApp extends React.Component {
  constructor(props) {
    super(props);
    this.state = { items: [], text: '' }; // ban đầu items ko có j hết (items gồm text và id), còn text kia là để tạm
    this.handleChange = this.handleChange.bind(this);
    this.handleSubmit = this.handleSubmit.bind(this);
  }

  render() {
    return (
      <div>
        <h3>TODO</h3>
        <TodoList items={this.state.items} />
        <form onSubmit={this.handleSubmit}>   // lúc bấm button submit
          <label htmlFor="new-todo">
            What needs to be done?
          </label>
          <input
            id="new-todo"
            onChange={this.handleChange}   // lúc nhập vô ô nhập nè
            value={this.state.text}
          />
          <button>
            Add todo {this.state.items.length + 1}
          </button>
        </form>
      </div>
    );
  }

  handleChange(e) {
    this.setState({ text: e.target.value });
  }

  handleSubmit(e) {
    e.preventDefault();
    if (this.state.text.length === 0) { // nếu ô nhập rỗng thì k làm gì hết
      return; 
    }
    const newItem = {
      text: this.state.text,
      id: Date.now()
    };
    this.setState(state => ({
      items: state.items.concat(newItem), // concat là tạo mảng mới từ mảng cũ và mảng mới
      text: ''
    }));
  }
}

class TodoList extends React.Component {
  render() {
    return (
      <ul>
        {this.props.items.map(item => (
          <li key={item.id}>{item.text}</li>
        ))}
      </ul>
    );
  }
}

ReactDOM.render(
  <TodoApp />,
  document.getElementById('todos-example')
);
```


Code tùm lum đọc vô không hiểu gì hết, hãy ngồi xuống ún trà rồi từ từ phân tích nào.

Trong TodoApp, ta còn thấy component TodoList đơn giản hơn nên ta xem nó trước. 

Đây đc gọi là **Stateless Component:** component chỉ có `props`, ko có `state` trong đó:

Chức năng của TodoList khá đơn giản *(vì stateless nên dễ quản lí)*: truyền `props` vào và tạo ra unordered list từ đó. 

*Cách hoạt động:*

Khi gõ vào ô nhập, hàm `handleChange` đc gọi (từ thẻ `input onChange`). Lúc này state bị thay đổi (ở đây sẽ thắc mắc state của TodoApp hay TodoList, lát nữa sẽ giải thích ở phần dùng bind) nên gọi hàm `render()`. 

Khi ấn vô nút Add thì button tự động gọi `onSubmit` ⇒ gọi `handleSubmit`

- Lớp cha thay đổi đc `props` lớp con. *(TodoList ko tự thay đổi `props` của mình đc mà TodoApp là lớp cha thì thay đổi đc.)*
- Render lớp cha thì render luôn lớp con (Virtual DOM).


> Đầu tiên viết `render()` trước, khi trạng thái thay đổi thì `setState` 

⇒ khi `state` của TodoList thay đổi thì tự nhiên `props` TodoApp thay đổi luôn ⇒ tự gọi `render()` của TodoApp.

- ***Tại sao phải sử dụng hàm bind()?** do cách hoạt động kì quái của* this *trong JS.*

C/C++: `this` trỏ đến class đc khai báo

JS: phụ thuộc lúc gọi hàm, tức là nơi nào gọi nó (chứ ko phải lúc khai báo)

```jsx
this.handleChange = this.handleChange.bind(this);
```

tạo ra hàm `handleChange` đồng thời cột cái `this.handleChange` với cái this của TodoApp *(tức là của React.Component, thì mới có setState)*

- **arrow function** tự động bind this dùm mình luôn.

- **class component** còn có thể viết thành **function component (ko có this)**.
